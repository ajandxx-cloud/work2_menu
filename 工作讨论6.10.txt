## 2. 对“attention 机制”的判断

**attention 可以加，但要加在正确位置。** 你的直觉是“让每个客户面对菜单时知道菜单之间的相互关系”，这本质上是要突破普通 MNL 的独立效用假设。普通 MNL 中，每个方案的系统效用主要由自身属性决定，菜单中其他方案只通过 softmax 分母影响选择概率；它不能很好表达“两个时间窗很接近所以互相替代”“某个中间选项让另一个选项显得更有吸引力”“菜单太多导致选择负担”等关系。

因此，attention 最适合放在**乘客选择模型或菜单价值评估模型**中，而不是简单放在 cost predictor 里。当前已有研究也在用 self-attention 建模 choice，并明确指出其目标是把现代神经网络结构引入选择建模，以捕捉传统模型难以表达的选择关系。([arXiv][1])

你可以这样写：

[
x_b=[d_b,\Delta_b,\hat \iota_b,p_b,\hat c_b,q_b,\text{type}_b]
]

[
h_b=\text{SelfAttention}(x_b,{x_j:j\in \mathcal M})
]

[
U_b=\theta^\top h_b+\epsilon_b
]

[
P(b|\mathcal M)=\frac{\exp(U_b)}{\exp(U_0)+\sum_{j\in\mathcal M}\exp(U_j)}
]

这样，attention 表示的就是：**某个服务选项的吸引力不仅取决于自身，还取决于它和同一菜单中其他选项的关系**。这比“在网络里加 attention 层”更有理论解释力。

但我建议你不要把 Work2 写成“我们加入 attention，所以更好”。更稳妥的主张是：

> 我们提出 DRT service bundle assortment optimization 框架，并进一步设计 attention-based choice/scoring module 来刻画菜单内服务选项之间的替代关系。

这样，即使 attention 的提升不是特别大，你的主贡献仍然成立。

# 5. 几个同时满足“可实现”和“有贡献”的研究方案

## 方案一：鲁棒时间窗服务菜单优化，最推荐作为主论文

**题目方向：**
考虑 ETA 不确定性的 DRT 时空服务菜单优化。

**核心思想：**
现在你的 work2 结果已经说明 ETA filtering 很关键。当前文稿结论中也写到，ETA filtering 是最有影响的层，正式解释 pricing 或 welfare 之前，应先 audit filtering 和 approximation quality。 你的修改建议文件也指出，当前贡献更像 diagnostic paper，最有价值的发现是 ETA-based candidate filtering 会显著改变菜单结构甚至导致 home-only menu collapse。

所以不要简单“关掉 ETA filter”，而是升级成：

[
\Pr(\tau_b\le t_b^{hi})\ge 1-\rho
]

或者使用 prediction interval：

[
[\hat\tau_b-z_\alpha\sigma_b,\hat\tau_b+z_\alpha\sigma_b]\cap [t_b^{lo},t_b^{hi}]\neq \emptyset
]

**贡献点：**
你可以说现有 DRT 菜单设计通常把时间窗可行性当作硬规则，但在在线 DRT 中 ETA 是预测量，硬筛选会误删可行方案。因此本文提出 uncertainty-aware service menu design，把时间窗预测误差显式纳入菜单生成与展示决策。

**实现方式：**
在当前 `hard / calibrated / interval / none` ETA filter 基础上扩展，不需要重写系统。你已有 `_choose_display_window`、ETA variant、false-negative pruning 记录。 只需要把它系统化成论文方法，并把 hard filter、calibrated filter、interval-overlap filter、no filter、robust filter 做成主实验。

**实验设计：**
比较 full display、nearest、top-k cheapest、passenger utility、expected profit menu、hard filter menu、robust filter menu。指标包括 net profit、acceptance rate、non-home uptake、opt-out rate、home-only menu rate、false-negative pruning、walking distance、pickup deviation、route cost。

**优先级：最高。**
这是最容易在现有代码上完成、也最容易形成独立贡献的方向。

## 方案二：Attention-based menu-aware choice model，用 attention 解释菜单内相互关系

**题目方向：**
基于注意力机制的 DRT 服务菜单选择模型。

**核心思想：**
传统 MNL 假设每个服务项效用相对独立，不能表达“菜单之间的相互关系”。你可以提出 menu-aware utility：

[
h_b=\text{Attn}(x_b,{x_j:j\in \mathcal M})
]

[
U_b=\theta^\top h_b+\beta p_b
]

其中 (x_b) 包含步行距离、等待时间、时间窗偏差、车内时间、价格、预测成本、剩余容量和是否 home option。attention 权重可以解释为：一个选项在多大程度上受其他选项影响。

**贡献点：**
它可以解决 MNL 的 IIA 局限，让服务菜单中的替代关系、相似时间窗、相近接驳点、价格锚定效应进入选择模型。已有 choice modeling 研究已经把 self-attention 用于选择建模，并强调 attention 可以作为 choice model 的神经结构。([arXiv][1])

**实现方式：**
不要一开始替换整个系统。先做一个独立模块：

输入：一个菜单 (\mathcal M) 的候选特征矩阵 (K\times F)。
输出：每个 offer 的 utility 或 choice probability。
训练数据：用你当前 MNL 生成 simulated choice label，或者用当前策略日志生成 accepted offer / opt-out label。
对比模型：MNL、MLP utility、attention utility。
最后把 attention choice probability 接入菜单评估函数。

**风险：**
如果没有真实乘客选择数据，attention 学到的是“仿真偏好”，不能声称验证真实行为。你应该把贡献表述为“attention-based behavioral mechanism under simulated heterogeneous preferences”，而不是“真实乘客偏好估计”。

**优先级：中高。**
适合作为方案一之后的增强模块。若时间有限，attention 做 ablation，不做主贡献。

## 方案三：Decision-focused menu learning，把 Work1 的 SPO 自然扩展到 Work2

**题目方向：**
面向 DRT 服务菜单选择的决策导向学习。

**核心思想：**
Work1 的优势是 SPO/decision-focused learning。Work2 如果只是 heuristic menu selection，和 Work1 的连接不够强。你可以把 predictor 从“预测成本/ETA”升级为“为菜单决策服务的预测器”。

当前文稿中已经有 shared predictor，并且会预测 (\hat c(b))、(\hat\tau_b)、(\hat\iota_b)，再用 frozen evaluation 隔离菜单政策效果。 但目前主要还是 predict-then-optimize。你可以定义 menu regret：

[
L_{\text{menu-regret}}
======================

\Pi(\mathcal M^*(c),c)-\Pi(\mathcal M^*(\hat c),c)
]

其中 (\mathcal M^*(\hat c)) 是用预测成本选出的菜单，(\mathcal M^*(c)) 是用真实/后验成本选出的菜单。

**贡献点：**
这会把 Work1 的“决策导向学习”延续到 Work2：Work1 是 cost prediction for pricing，Work2 是 cost/time prediction for menu selection。比单纯 attention 更符合你的博士论文主线。

**实现方式：**
小候选集用 exact enumeration 产生 oracle menu，训练 predictor 或 scoring model 减小 menu regret。大候选集用 greedy 或 sampled subset 近似。当前方法已经有 exact enumeration for small candidate sets 和 greedy forward selection for large candidate sets。 当前文稿也已经承认 exact 只是 surrogate benchmark，大规模需要 greedy 近似。

**风险：**
SPO-style loss 比较难写得稳定。建议先做“oracle label learning”：对每个 request 生成 exact optimal menu，把问题变成 learning-to-rank / pairwise ranking。跑通后再升级为 differentiable regret。

**优先级：高。**
如果你想让 Work2 和 Work1 形成强连续性，这是最好的理论路线。

## 方案四：服务质量约束下的菜单优化，保证不只追求利润

**题目方向：**
考虑接受率与服务公平约束的 DRT 服务菜单优化。

**核心思想：**
你当前结果里一个风险是：某些菜单策略可能提高 profit，但降低 acceptance 或 non-home uptake。当前 results 也已经提醒，profit improvement 不自动意味着 service-quality improvement。

因此可以把问题改成：

[
\max_{\mathcal M_r}\sum_{b\in\mathcal M_r}P_{rb}(p_b-\hat c_b)
]

s.t.

[
P_{r0}(\mathcal M_r)\le \bar q
]

[
\mathbb E[d_b|\mathcal M_r]\le \bar d
]

[
\mathbb E[|\Delta_b||\mathcal M_r]\le \bar \Delta
]

或者加惩罚：

[
\max ; \text{profit}
-\lambda_0P_{0}
-\lambda_d\mathbb E[d]
-\lambda_\Delta\mathbb E[|\Delta|]
]

**贡献点：**
这比单纯“利润最大化菜单”更适合公共交通。你可以强调 DRT 不是纯商业平台，还要兼顾服务覆盖、乘客体验和公共交通属性。近年的 constrained assortment optimization 也在研究带覆盖/服务约束的 MNL assortment，这说明“约束型菜单优化”本身有明确的优化理论背景。([arXiv][2])

**实现方式：**
你 GitHub 里已经有 `service_constrained_expected_profit`、`risk_adjusted_expected_profit`、`service_guarded_expected_profit` 等策略雏形。 代码中也已经有 outside probability guardrail 和 service quit penalty。 这说明实现成本很低，只需要把它从“策略变体”提升为“正式模型”。

**优先级：高。**
特别适合作为 Work2 的管理学/公共交通贡献。

下面给出 **方案1的详细执行方案**：

---

## 1. 研究问题与目标

**目标**：在多对一 DRT 场景下，将服务菜单定义为“接驳点 × 时间窗 × 价格”组合的候选集，并为每位乘客选择最优展示菜单，使系统期望净收益最大化，同时保证乘客体验。

* 菜单决策变量：(\mathcal M_r \subseteq \mathcal B_r)

* 约束：

  * 非 home bundle 数量 (|\mathcal M_r \setminus {b^0}| \le k)
  * ETA 可行性：(\Pr(\tau_b \in [t_b^{lo}, t_b^{hi}]) \ge 1-\rho)
  * 外部选项概率上限：(P_{r0}(\mathcal M_r) \le \bar q)

* 目标函数：最大化系统期望净收益
  [
  \Pi = \sum_{r \in \mathcal R} \sum_{b \in \mathcal M_r} \pi(b|\mathcal M_r) (p_b - c(b))
  ]

其中，(\pi(b|\mathcal M_r)) 为乘客选择概率，由 MNL 模型给出。

---

## 2. 技术路线

### 2.1 候选菜单生成

1. 对每位到达乘客生成候选上车点集合。
2. 对每个候选点生成时间窗 (w = [t_b^{lo}, t_b^{hi}])，并根据步行可达性和车辆状态筛除不可行选项。
3. 每个候选组合形成完整 offer：
   [
   \text{offer}_j = (m_j, w_j, p_j)
   ]
4. 价格 (p_j) 由 Work1 的边际运营成本预测和解析式最优定价生成。

---

### 2.2 乘客选择建模

* 采用 MNL 模型，考虑乘客对：

  * 步行距离 (d_b)
  * 预计车内时间 (\hat \iota_b)
  * 时间窗偏差 (\Delta_b = \hat\tau_b - t_r^{req})
  * 价格 (p_b)
* 乘客异质性：价格、步行和 IVT 敏感度不同。
* 外部选项作为额外备选方案。

---

### 2.3 菜单优化方法

**核心算法**：分层求解

1. **Exact Enumeration（小规模候选集）**：

   * 枚举所有可能的菜单组合 (|\mathcal B_r| \le \text{menu_exact_threshold})
   * 计算每个菜单的期望收益，选择最优菜单。

2. **Greedy Forward Selection（大规模候选集）**：

   * 初始化菜单：包含 home bundle。
   * 迭代加入能最大化 (\widehat \Pi(\mathcal M)) 的非 home offer，直到到达 cap (k)。
   * 记录 exact-vs-greedy gap，用于验证近似质量。

3. **ETA 不确定性处理**：

   * 对每个候选菜单应用 robust ETA filter 或 interval-based filtering，保证时间窗可行性。
   * 记录 false-negative pruning 率和菜单覆盖率。

4. **联合优化**：

   * 对每个候选菜单计算 system-aware cost：
     [
     \hat c_\text{sys}(b) = \hat c(b) + \mu_r \cdot \rho_b + \mu_q / (q_b + \epsilon)
     ]
   * 定价后计算期望收益：
     [
     \widehat \Pi(\mathcal M) = \sum_{b \in \mathcal M} \pi(b|\mathcal M) (p_b - \hat c_\text{sys}(b))
     ]

---

### 2.4 实验设计

1. **小规模合成数据**：

   * 200 个地理位置，10 个候选接驳点，90 个乘客，车队 12 辆。
   * 比较 DRPO 与基线策略：

     * Only-home / Only-meeting-points / No-pricing / Static-pricing / DSPO
   * 指标：系统净收益、乘客上车选择率、opt-out rate。

   * 燕郊通勤场景，300 个家庭出行点，车队 20 辆。
   * 比较 DRPO 与同样基线策略。
   * 记录上门接送比例、拒绝率、平均乘车时间、平均步行距离。

3. **敏感性分析**：

   * 调整时间窗宽度、乘客偏好异质性。
   * 分析 robust ETA filter 对菜单选择的影响。
   * 记录 exact vs greedy 差异。
