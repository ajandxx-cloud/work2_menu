# 二、时间窗如何进入模型（核心建模）

## 2.1 服务产品定义（Work2核心）

每个候选服务不再只是：

[
m \in M
]

而变成一个三元组：

[
j = (m, w, p)
]

其中：

* (m)：接驳点（meeting point）
* (w = [t^{start}, t^{end}])：时间窗
* (p)：价格

👉 这一步的关键意义：

> 你把“空间选择问题”扩展成“时空产品设计问题”

---

## 2.2 时间窗如何生成（非常关键）

你必须定义一个“候选时间窗生成机制”，否则模型不可用。

### 方法A（推荐，论文最稳）

基于车辆预计到达时间：

[
\hat{t}_{m}
]

生成：

[
w_k = [\hat{t}_m - \Delta_1^k,\ \hat{t}_m + \Delta_2^k]
]

例如：

* 严格窗口：[-2min, +2min]
* 中等窗口：[-5min, +5min]
* 宽松窗口：[-10min, +10min]

👉 这样你得到：

[
\mathcal{W}_m = {w_1, w_2, ..., w_K}
]

最终候选集合：

[
\mathcal{O}*i = {(m, w_k, p*{mk})}
]

---

## 2.3 为什么时间窗很关键？

时间窗引入后，你的模型多了一个核心权衡：

| 变量    | 影响           |
| ----- | ------------ |
| 更宽时间窗 | 更高接受率，但运营更难  |
| 更窄时间窗 | 更低不确定性，但拒单增加 |

👉 这直接变成经典：

> **revenue vs service reliability trade-off**

# 三、乘客选择模型如何扩展（关键）

你现在不能只用：

[
U = \beta_{walk} + \beta_{price} + \beta_{ivt}
]

必须加入时间窗惩罚。

---

## 3.1 新效用函数（建议标准写法）

[
U_{i j} =

* \beta^{walk} d_{ij}
* \beta^{price} p_{ij}
* \beta^{ivt} \tau_{ij}
* \beta^{time} \cdot TD_{ij}

- \varepsilon_j
  ]

其中新增：

### 🔴 时间窗不适配惩罚（核心）

[
TD_{ij} = \max(0, t_i - w_j^{end}) + \max(0, w_j^{start} - t_i)
]

解释：

* 提前到 → 等待成本
* 迟到 → 错过成本（强惩罚）

---

## 3.2 外部选项

仍然保留：

[
U_{i0} = \beta_0
]

表示：

* 不接受任何方案
