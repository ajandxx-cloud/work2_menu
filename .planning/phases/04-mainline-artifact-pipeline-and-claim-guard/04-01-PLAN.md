when clean, and formal requires dependency snapshot plus loaded checkpoint
provenance.

## Task 1 - Tighten Tier-Specific Claim Gates

**Type:** code
**Files:**
- `work2_coding/Src/artifact_status.py`

**Action:**
1. Update `classify_artifact()` so smoke-tier rows never return
   `status="claim_ready"`; clean smoke rows should classify as `diagnostic`
   with a clear reason.
2. Preserve pilot claim-ready behavior when rows are completed,
   non-placeholder, non-diagnostic, non-filter-only, and checkpoint requirements
   pass.
3. Ensure formal claim-ready requests require a dependency snapshot.
4. Ensure formal rows with `checkpoint_required=true` require
   `checkpoint_load_status="loaded"` for claim-ready evidence. Do not allow
   `not_requested` to satisfy formal checkpoint provenance.
5. Keep existing blockers for failed, blocked, incomplete, contract-only,
   placeholder-only, diagnostic-only, no-filter-only, and bad-checkpoint rows.

**Verify:**
```powershell

python scripts/test_artifact_gates.py
```

**Acceptance Criteria:**
- Smoke classification is diagnostic, not claim-ready.
- Pilot synthetic clean rows can classify as claim-ready.
- Formal synthetic rows require dependency snapshot and loaded checkpoint status.
- Existing blocked/incomplete/diagnostic cases still block or downgrade.
