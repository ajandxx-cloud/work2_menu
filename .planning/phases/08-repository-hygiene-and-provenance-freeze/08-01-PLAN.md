# Phase 08 Plan: Repository Hygiene And Provenance Freeze

**Status:** Planned
**Created:** 2026-06-11
**Mode:** MVP

## Goal

Reduce provenance risk before checkpoint training and new empirical evidence by classifying current repository state, tightening ignore policy, and recording the artifact/provenance gate for Phase 09+.

## Requirements

- PROV-01: classify dirty and untracked files.
- PROV-02: protect the repository from accidental venv/cache/temp/raw artifact tracking.
- PROV-03: document generated artifacts to track versus keep local.
- PROV-04: define how evidence runs report clean or narrowly documented dirty provenance.

## Scope

In scope:
- Add root-level ignore policy for repository-wide caches, virtual environments, temporary files, LaTeX byproducts, raw local outputs, checkpoint binaries, and generated planning reports.
- Preserve `work2_coding/.gitignore` as the runtime-local ignore layer.
- Classify current `git status --short --ignored` output into planning docs, generated artifacts, local outputs, dependency files, deleted user documents, and other local state.
- Document that `.planning/reports/MILESTONE_SUMMARY-v2.0.md` is a local generated planning report, not canonical evidence.
- Record an evidence cleanliness gate for Phase 09+.
- Preserve the user's claim-honesty rule: unsupported attention results must be marked failed or blocked.

Out of scope:
- Training checkpoints.
- Running pilot or formal evidence.
- Editing generated result rows or paper artifacts by hand.
- Deleting local files discovered during classification.

## Work Items

1. Create or update root `.gitignore`.
   - Ignore Python caches, venvs, editor files, temp files, LaTeX build byproducts, raw output roots, checkpoint binaries, and generated planning reports.
   - Do not ignore review-facing `artifacts/`, `work2_coding/artifacts/`, `.planning/phases/`, or canonical planning docs.

2. Record hygiene and artifact policy.
   - Create `08-HYGIENE-REPORT.md`.
   - Classify current untracked/ignored state.
   - Explicitly classify `.planning/reports/MILESTONE_SUMMARY-v2.0.md`.
   - Record track-vs-local generated artifact rules.

3. Record provenance and claim gate.
   - Evidence runs should aim for `git_dirty=false`.
   - If `git_dirty=true`, the reason must be narrow and documented in run metadata/status.
   - Results that do not support attention-improves-DSPO must be marked failed or blocked, not softened.

4. Verify.
   - Run import smoke from `work2_coding/`.
   - Run relevant provenance/artifact contract tests.
   - Run `git status --short --ignored` and confirm only documented ignored local state remains.

## Acceptance Criteria

- `PROV-01` has a command-backed classification table.
- `PROV-02` is satisfied by root `.gitignore` plus existing runtime-local ignore policy.
- `PROV-03` is documented in Phase 08 outputs.
- `PROV-04` is documented as a Phase 09+ evidence gate.
- Phase 08 verification records exact commands and pass/fail results.
