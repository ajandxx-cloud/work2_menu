# Phase 09 Plan: Shared Checkpoint Training Pipeline

**Status:** Planned
**Created:** 2026-06-11
**Mode:** MVP

## Goal

Create real shared pilot/formal predictor checkpoints with explicit provenance so attention evidence runs are not blocked by missing checkpoint files.

## Requirements

- CKPT-01: stable shared checkpoint training entry point exists.
- CKPT-02: pilot/formal checkpoints are real trained weights, not placeholders.
- CKPT-03: sidecars record sha256, command, seed, split, dataset, run_id, git provenance, training args, and timestamp.
- CKPT-04: checkpoints load through the same code path used by `run_study.py`.
- CKPT-05: missing, mismatched, or random-weight checkpoints are refused for pilot/formal evidence.

## Work Items

1. Add `scripts/train_shared_checkpoint.py`.
   - Load a study manifest.
   - Resolve the declared shared checkpoint path.
   - Build a predictor architecture compatible with the manifest's `algo_name` and parser defaults.
   - Train deterministic supervised weights.
   - Save `supervised_ml.pt` and a JSON sidecar.

2. Add a focused test script.
   - Train into a temporary output root.
   - Assert `.pt` and sidecar exist.
   - Assert sidecar hash matches the checkpoint file.
   - Assert weights changed from deterministic initialization.
   - Assert `model.load_checkpoint(...)` returns `loaded`.
   - Assert missing/mismatched checkpoint tests still fail closed.

3. Generate local pilot and formal checkpoints.
   - `python scripts/train_shared_checkpoint.py --study pilot_attention_dspo`
   - `python scripts/train_shared_checkpoint.py --study formal_attention_dspo`

4. Verify.
   - Import smoke.
   - New shared checkpoint training test.
   - Existing checkpoint provenance test.
   - Attention manifest contract test.
   - Pilot actual replay smoke with max two policies, proving the pilot checkpoint can load through `run_study.py`.

## Acceptance Criteria

- Both manifest-declared attention checkpoints exist locally.
- Both sidecars record required provenance.
- Tests prove load status is `loaded` for generated checkpoints.
- Missing/mismatched checkpoint refusal behavior remains covered.
- Phase 09 docs state that checkpoint creation does not itself validate the attention-improves-DSPO claim.
