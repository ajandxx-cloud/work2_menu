## Implementation Decisions

### Outside-Option Parameterization
- Add `outside_option_util` as a constructor parameter to CustomerChoiceModel, defaulting to 0.0.
- Replace hardcoded `utils[n_offers] = 0.0` with `utils[n_offers] = self.outside_option_util`.
- Expose the parameter in the YAML study schema so studies can scan u_0 values.
- Backward-compatible: default 0.0 means all existing studies continue to work unchanged.

### Literature Anchoring
- Add 3-4 DRT/SP literature references to references.bib.
- Anchor three parameters: price sensitivity (beta), IVT weight (alpha-tau), and outside-option calibration.
- Present as a "plausibility range" paragraph citing literature bounds, noting where simulator values fall within or outside those ranges.
- Update method.tex Table 2 "Source" column from "Simulator design parameter" to "Literature-bounded design parameter" with a footnote citing ranges.

### Experiment Design
- Scan 5 outside-option utility points: u_0 = -1.0, -0.5, 0.0, +0.5, +1.0 (symmetric around current default).
- Run on primary RC only — focused scan sufficient for reviewer's request.
- Use same 3-policy set as main results: strict-filter, no-filter, uniform-markdown.
- Reuse existing phase 27 MNL sensitivity results — they already cover base_util/home_util/incentive_sens regimes.

### Manuscript Framing
- Explicit section stating "this is literature-bounded sensitivity testing, not external validation with revealed preferences."
- Extend results.tex MNL section by adding a subsection "Outside-Option Sensitivity" after the existing MNL section.
- Remove the "deferred to future work" language about outside-option scanning — replace with actual findings.
- Keep honest about what the scan does and does not prove.

### Claude's Discretion
