# Plan 03-01: Conservative Split-Level Statistical Presentation

## Goal

Rewrite the statistical presentation in results.tex and relabel the uncertainty table to make the inference scope crystal clear: RC has 6 split pairs (mechanism diagnostic), Austin/Seattle have 2 split pairs each (descriptive only, no interval claims).

## Tasks

### Task 1: Relabel policy_gap_uncertainty_summary.tex via build_artifacts.py

**File:** `ooh_code/scripts/build_artifacts.py`

The existing uncertainty table title already says "split-level paired uncertainty summary" and distinguishes "split bootstrap interval" vs "observed range" as interval types. The key issue is the word "uncertainty" in the title and the bootstrap language for RC.

Actions:
- Change the table caption from "split-level paired uncertainty summary" to "Split-level paired gap summary"
- For RC rows, change "Split bootstrap interval" to "Split-level range (6 pairs)"
- For Austin/Seattle rows, change "Observed range" to "Two-pair descriptive range"
- Ensure the caption states: "Austin and Seattle each have only two split pairs; their gaps are reported as descriptive checks, not stable interval estimates."
- Rename column "Interval type" to "Range type"

After modifying build_artifacts.py, regenerate the table by running the build script or reconstructing it manually.

**File:** `ooh_code/manuscript/sections/results.tex`

**File:** `ooh_code/manuscript/sections/results.tex`
