**Files:** `ooh_code/scripts/build_artifacts.py`, new `ooh_code/artifacts/tables/profit_decomposition_summary.tex`

### Task 4: Add consumer surplus formula to appendix

**File:** `ooh_code/manuscript/sections/appendix.tex`

Add a new subsection "Consumer Surplus Computation" that:
- Defines the MNL-derived consumer surplus for a single request: CS_i = (1/alpha_price) * [ln(exp(V_chosen) + exp(V_outside))] when a menu bundle is chosen, or CS_i = 0 when the outside option is chosen (surplus normalized to zero)
- Defines all-request average surplus: (1/N) * sum(CS_i) for all N requests
- Defines accepted-user average surplus: (1/N_accepted) * sum(CS_i) for accepted requests only
- Notes: "The all-request surplus includes users who chose the outside option and received zero surplus by construction. The accepted-user surplus averages only over requests where a menu bundle was selected. These two measures differ most when opt-out rates are high."
- References the existing welfare table for reported values

### Task 5: Update managerial.tex with tradeoff narrative

**File:** `ooh_code/manuscript/sections/managerial.tex`

Add a paragraph to the existing managerial interpretation:
- "The profit decomposition (Table~\ref{tab:profit_decomposition}) reveals that menu-optimization gaps are driven by multiple cost channels simultaneously, not by a single dominant factor. When evaluating whether to deploy a filtering or pricing rule, operators should consider: (1) the net-profit direction and magnitude, (2) whether acceptance rates change, (3) whether passenger-facing service metrics (walk distance, pickup deviation, in-vehicle time) change, and (4) whether the total volume of served requests shifts."
- Connect to the dual-surplus framing: "The all-request consumer surplus and accepted-user consumer surplus can move in opposite directions when a policy changes the acceptance rate — a profit improvement paired with lower acceptance does not automatically indicate better passenger outcomes."

### Task 6: Update limitations.tex with surplus caveat

**File:** `ooh_code/manuscript/sections/limitations.tex`

Add to the existing MNL parameter paragraph or as a new paragraph:
- "Consumer surplus is computed under the implemented MNL utility specification (Appendix, Consumer Surplus Computation). The all-request surplus normalizes outside-option surplus to zero, while the accepted-user surplus averages only over requests where a menu bundle was selected. Because the MNL parameters are fixed design choices rather than estimated from passenger data, the surplus values should be interpreted as internally consistent model outputs, not as welfare measurements from revealed preferences."
- Add: "When comparing policies, a change in acceptance rate shifts the composition of accepted users: if a filter removes low-value requests, the accepted-user surplus can increase even if no individual passenger's experience improves. This selection effect is inherent in the menu-optimization framework and should be considered when interpreting surplus differences across policies."
