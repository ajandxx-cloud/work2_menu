  def _select_menu_candidates(self, customer, candidates):
      home_offer, ooh_candidates = self._split_menu_candidates(candidates)
      if self.menu_policy == "offer_all_feasible_bundles":
          menu = list(candidates)
      elif self.menu_policy == "nearest_heuristic":
          selected = sorted(ooh_candidates, key=lambda o: (o.walk_distance, o.time_deviation))[:self.menu_k]
          menu = ([home_offer] if home_offer is not None and self.menu_keep_home else []) + selected

Implement 3 new DRT baseline strategies in DSPO_Menu.py `_select_menu_candidates` method, register them in parser.py, and create the study manifest.

**Strategy 1: insertion_cost_greedy**
- Sort ooh_candidates by `offer.predicted_cost` ascending (cheapest insertion first)
- Take top `self.menu_k` candidates
- This is identical in behavior to `top_k_cheapest` -- WAIT. Per the user decision, insertion-cost greedy should use "real-time cheapestInsertionCosts". The existing `top_k_cheapest` sorts by `offer.predicted_cost` which IS the insertion cost. So insertion_cost_greedy is actually the same sorting logic but with a different display label for the operational-baseline narrative context.
- However, to differentiate: make insertion_cost_greedy sort by `offer.predicted_cost` (same key) but it will appear as a separate policy in the manifest so it gets its own row in the results table. The distinction is editorial: "insertion cost greedy" frames the baseline as a dispatch heuristic, while "top_k_cheapest" was originally a menu-policy comparison.
- Implementation: add `elif self.menu_policy == "insertion_cost_greedy":` block that sorts by predicted_cost, takes top menu_k, prepends home_offer.

**Strategy 2: min_lateness**
- Sort ooh_candidates by `offer.time_deviation` ascending (minimum pickup time deviation first)
- Take top `self.menu_k` candidates
- Prepend home_offer if present and keep_home is True
- Implementation: add `elif self.menu_policy == "min_lateness":` block
  ```python
  selected = sorted(ooh_candidates, key=lambda offer: offer.time_deviation)[:self.menu_k]
  menu = ([home_offer] if home_offer is not None and self.menu_keep_home else []) + selected
  ```

**Strategy 3: random_top_k**
- Randomly sample `self.menu_k` candidates from ooh_candidates without replacement
- Provides a floor baseline that makes other strategies' performance interpretable
- Use `import random` (already available via numpy, but use random.sample for clarity)
- Implementation: add `elif self.menu_policy == "random_top_k":` block
  ```python
  import random
  k = min(self.menu_k, len(ooh_candidates))
  selected = random.sample(ooh_candidates, k=k)
  menu = ([home_offer] if home_offer is not None and self.menu_keep_home else []) + selected
  ```
- NOTE: For reproducibility, the random seed should be set from `self.config.seed` or similar. Check how the existing codebase handles randomness. The study manifest's `seed` parameter controls the experiment-level seed, so per-episode randomness should be fine.

```python
def build_operational_baselines_artifacts(rows, prefix="operational_baselines"):
    order = {
        "full_display": 0,
        "menu_optimization": 1,
        "insertion_cost_greedy": 2,
        "min_lateness": 3,
        "random_top_k": 4,
    }
    rows = sorted(rows, key=lambda row: (order.get(row["variant_tag"], 99), row["variant_tag"]))
    table_rows = [
        [
            row["display_label"],
            format_metric(row.get("net_profit")),
            format_metric(row.get("mean_net_profit_gap_vs_reference")),
            format_metric(row.get("net_profit_win_rate_vs_reference")),
            format_metric(row.get("acceptance_rate")),
            format_metric(row.get("average_menu_size")),
            format_metric(row.get("avg_meeting_point_count_per_menu")),
        ]
        for row in rows
    ]
    write_tex_table(
        ARTIFACTS_DIR / "tables" / f"{prefix}_summary.tex",
        caption="Operational baseline comparison on the RC low-uptake benchmark. Insertion-cost greedy, minimum-lateness ranking, and random-top-k provide dispatch-floor and greedy-policy context for the menu-optimization heuristic.",
        label=f"tab:{prefix}",
        headers=[
            "Policy",
            "Mean net profit",
            "Gap vs full",
            "Win rate vs full",
            "Acceptance",
            "Menu size",
            "Meeting-point count",
        ],
        rows=table_rows,
    )
    write_csv(ARTIFACTS_DIR / "tables" / f"{prefix}_summary.csv", rows)
```
