## Test Structure

**Suite Organization:**
```yaml
# ooh_code/experiments/studies/smoke_rc.yaml
name: smoke_rc
type: policy_compare
reference_policy: offer_all_feasible_bundles
base_args:
  load_data: false
  max_episodes: 1
  eval_episodes: 1
  max_steps_r: 1
  max_steps_p: 0.5
  n_vehicles: 2
  veh_capacity: 2
policies:
  - menu_optimization
  - nearest_heuristic
  - top_k_cheapest
splits:
  - id: smoke_0_to_1
    train_split: 0
    test_split: 1
```

**Patterns:**
- Use `smoke_rc` for quick end-to-end verification. It avoids bundled data by setting `load_data: false` and keeps training/evaluation small in `ooh_code/experiments/studies/smoke_rc.yaml`.
- Use paired frozen evaluation for research comparisons: train one shared predictor, generate request traces, then evaluate multiple display policies against the same traces. This is implemented in `execute_study_manifest` in `ooh_code/Src/research_pipeline.py` and documented in `ooh_code/docs/WORK2_EXPERIMENT_PROTOCOL.md`.
- Validate study manifests through execution. `load_manifest` and `parser_namespace_with_overrides` in `ooh_code/Src/research_pipeline.py` reject unresolved manifests, non-mapping YAML, and unknown parser arguments.
- Validate manuscript wiring with `ooh_code/scripts/check_manuscript.py`, which scans `ooh_code/manuscript/*.tex` and `ooh_code/manuscript/sections/*.tex` for undefined internal references and missing artifact files.
- Validate artifact generation by running `ooh_code/scripts/build_artifacts.py --study <study_or_suite>` and checking outputs under `ooh_code/artifacts/results_snapshot/`, `ooh_code/artifacts/tables/`, `ooh_code/artifacts/figures/`, and `ooh_code/artifacts/RESULTS_SUMMARY.md`.

## Mocking

**Framework:** Not detected.

**Patterns:**
```python
# ooh_code/Src/research_pipeline.py
request_traces = generate_request_traces(
    base_args,
    training_metadata["checkpoint_path"],
    eval_episodes=base_args.eval_episodes,
    trace_seed=trace_seed,
)
```

```python
# ooh_code/run_menu_compare.py
solver.test_env.set_request_trace(request_traces)
...
solver.test_env.clear_request_trace()
```

**What to Mock:**
- Prefer replayed request traces over mocking the simulator. `generate_request_traces`, `set_request_trace`, and `clear_request_trace` provide deterministic inputs for paired policy comparisons in `ooh_code/Src/research_pipeline.py` and `ooh_code/run_menu_compare.py`.
- Use small manifest settings for smoke verification instead of mocking training or routing internals: `load_data: false`, `max_episodes: 1`, `eval_episodes: 1`, `n_vehicles: 2`, and `veh_capacity: 2` in `ooh_code/experiments/studies/smoke_rc.yaml`.
- Use `--skip_compile` for manuscript verification when LaTeX is unavailable rather than mocking compiler binaries; the fallback is built into `ooh_code/scripts/build_manuscript.py`.

**What NOT to Mock:**
- Do not mock `DSPO_Menu`, `Parcelpoint_py`, or `hygese` behavior for research conclusions. The project‚Äôs comparisons depend on the actual solver/simulator pipeline in `ooh_code/Src/Algorithms/DSPO_Menu.py`, `ooh_code/Environments/OOH/Parcelpoint_py.py`, and `ooh_code/Src/work2_runtime.py`.
- Do not mock normalized output schemas when changing artifact generation. Rebuild from real `study_summary.json` and `aggregate_variant_summary.json` through `ooh_code/scripts/build_artifacts.py`.
- Do not hand-edit generated files in `ooh_code/artifacts/tables/`, `ooh_code/artifacts/figures/`, or `ooh_code/artifacts/results_snapshot/` to satisfy checks; regenerate them from study outputs.

## Test Types

**Unit Tests:**
- Not used as a formal suite.
- The closest unit-level checks are deterministic helper failures in ordinary runtime paths: invalid manifest shape in `load_manifest`, unknown overrides in `parser_namespace_with_overrides`, unsupported study types in `variant_specs_for_manifest`, invalid booleans in `Parser.str2bool`, and invalid data paths in `Utils.load_demand_data`.
- When adding high-risk pure helpers to `ooh_code/Src/research_pipeline.py` or `ooh_code/scripts/build_artifacts.py`, add lightweight script-level validation or a future proper test module rather than relying only on full study runs.

**Integration Tests:**
- The primary integration check is `python scripts/run_study.py --study smoke_rc` from `ooh_code/`.
- A successful study run should produce a completed `study_summary.json`, `normalized_rows.json`, `normalized_rows.csv`, `aggregate_variant_summary.json`, `aggregate_variant_summary.csv`, split summaries, and per-variant metrics under `ooh_code/outputs/studies/smoke_rc/<run_id>/`.
- Artifact integration is checked with `python scripts/build_artifacts.py --study smoke_rc`, which writes snapshots, tables, figures, and `ooh_code/artifacts/RESULTS_SUMMARY.md`.
- Manuscript integration is checked with `python scripts/check_manuscript.py` and `python scripts/build_manuscript.py --skip_compile`. Full PDF compilation additionally requires `latexmk` or `pdflatex`.

**E2E Tests:**
- No browser or UI E2E framework is used.
- Research E2E flow:
```bash

python scripts/run_study.py --study smoke_rc
python scripts/build_artifacts.py --study smoke_rc
```
