# 6.4 实验重构方案：从插入成本表征转向 Choice-Aware Menu Optimization

## 0. 结论先行

现有 Work 2 结果不说明“实验完全失败”，但说明原始主线不适合作为论文核心贡献继续硬推：

> **CNN-SetMenuNet 的集合表征能力没有稳定转化为更好的 realized net profit。**

因此，Work 2 不应继续围绕“CNN-SetMenuNet 是否比 CNN 更会预测插入成本”展开，而应改为：

> **DRT 平台如何在乘客选择、动态定价、route-cost feedback 和 opt-out risk 下设计有限服务菜单，使平台收益与服务质量之间取得更稳健的平衡。**

新的论文主线是 **choice-aware / profit-aware dynamic service menu optimization**。CNN-SetMenuNet 保留为 baseline / diagnostic method，而不是主方法。

---

## 1. 为什么旧主线不够稳

原始逻辑链条是：

1. 预测候选 meeting point 的边际插入成本；
2. 选择预测成本最低的 Top-L 点；
3. 乘客从菜单中选择；
4. 系统净利润提高。

当前 formal evidence 显示这条链条存在目标错位：

- Top-L overlap / NDCG 可以很好，但 net profit 不一定好；
- Oracle Menu 没有形成清晰 profit upper bound；
- MLP / SetMenuNet 可能出现极高 quit rate，但 raw net profit 看起来不差；
- CNN-SetMenuNet 没有稳定超过 Cost-L heuristic 或 CNN-Menu。

这说明：

> **“低插入成本菜单”不等于“高系统收益菜单”。**

所以不能再把插入成本预测准确性当作 Work 2 的核心目标。真正需要优化的是菜单在乘客选择之后带来的期望系统贡献。

---

## 2. 新研究问题

### English framing

**Choice-aware dynamic service menu optimization for demand-responsive transit with passenger boarding-point choice.**

### 中文理解

考虑乘客上车点选择行为的 DRT 动态服务菜单优化。

### 新贡献结构

**贡献 1：问题贡献**

提出一个同时考虑有限菜单展示、乘客选择、动态定价、route-cost feedback 和 opt-out risk 的 DRT service menu optimization 问题。

**贡献 2：方法贡献**

提出 choice-aware / profit-aware 菜单优化方法，直接评估菜单的 expected system contribution，而不是间接预测候选点插入成本。

**贡献 3：实验贡献**

通过多 seed 仿真证明：仅基于插入成本的菜单设计可能无法改善 realized net profit；显式考虑乘客选择概率、退出风险和路径成本反馈的方法更可能获得稳健的 profit-service tradeoff。

---

## 3. 新目标函数

对每个展示菜单 \(M_t\)，定义 expected menu contribution：

\[
\mathbb{E}[\mathrm{profit}(M_t)]
=
\sum_{k \in M_t} P_k(M_t, p_t)
\left(\mathrm{fare}_{k} - \mathrm{expected\ route\ cost}_{k}\right)
- P_0(M_t, p_t) \cdot \eta_{\mathrm{quit}}
\]

其中：

- \(M_t\)：展示给乘客的 meeting-point 菜单；
- \(P_k(M_t,p_t)\)：乘客选择菜单项 \(k\) 的 MNL 概率；
- \(P_0(M_t,p_t)\)：outside option / quit 概率；
- \(\mathrm{fare}_{k}\)：动态定价后的收入；
- \(\mathrm{expected\ route\ cost}_{k}\)：选择该点后的预期路径成本；
- \(\eta_{\mathrm{quit}}\)：退出惩罚或机会损失。

第一版不需要新神经网络。由于默认 \(K=10, L=3\)，每个请求只有：

\[
\binom{10}{3}=120
\]

个 meeting-point 菜单，可以直接枚举。

---

## 4. 必须新增的服务约束

raw net profit 不能单独作为主指标。否则一个让大量乘客退出的策略可能因为“少服务、少成本”而看起来不错。

至少需要加入以下一种约束或指标：

### A. Adjusted profit

\[
\mathrm{AdjustedProfit}
=
\mathrm{NetProfit}
- \eta_q \cdot \mathrm{QuitRequests}
\]

### B. Service-constrained net profit

\[
\max \mathrm{NetProfit}
\quad
\mathrm{s.t.}
\quad
\mathrm{quit\ rate} \leq \bar q
\]

建议 pilot 先试：

- \(\bar q = 0.35\)
- \(\bar q = 0.40\)

### C. Multi-objective score

\[
\mathrm{Score}
=
\mathrm{NetProfit}
- \eta_q \cdot \mathrm{QuitRate}
- \eta_w \cdot \mathrm{AvgWalk}
- \eta_t \cdot \mathrm{AvgIVT}
\]

第一版推荐采用 A 或 B，避免一开始引入过多权重。

---

## 5. Oracle 必须拆分

旧 Oracle Menu 可能只是 **Cost Oracle**，不能作为 net profit upper bound。

| Oracle | 定义 | 作用 |
|---|---|---|
| Cost Oracle | 使用真实插入成本选 Top-L 菜单 | 成本排序上界 / 诊断 |
| Profit Oracle | 使用真实 expected profit 选菜单 | 新主目标的参考上界 |
| Realized Oracle | 事后知道乘客选择和路径结果的理论上界 | 可选诊断，v1 可不做 |

论文和表格中必须避免把 Cost Oracle 写成 profit oracle。

## 6. Phase 08：Work2 Redesign Pilot

### 目标

建立 choice-aware / profit-aware DRT service menu optimization 的最小可运行实验框架，验证新 objective 是否能缓解 high quit 和 profit mismatch。

### 保留内容

- 保留 Phase 06 / formal main 的所有结果；
- 不覆盖旧 CNN-SetMenuNet artifacts；
- CNN-SetMenuNet 保留为 baseline / diagnostic method；
- 不修改 Work 1 的 pricing、MNL choice、HGS route-cost core。

### 新增方法

1. **Expected-Profit Enumeration**
   - instance: RC
   - \(K=10\)
   - \(L=3\)
   - home option always shown
   - 枚举所有 \(C(10,3)=120\) 个 meeting-point 菜单
   - 计算每个菜单的 MNL choice probability、expected revenue、expected route cost、quit penalty
   - 选择 expected profit 最大的菜单

2. **Service-Constrained Expected-Profit**
   - 在 expected profit 最大化基础上加入 quit-rate guardrail；
   - 若没有菜单满足约束，需要输出 diagnostic fallback；
   - 不允许静默退化成高 quit-rate 策略。

3. **Profit Oracle**
   - 用真实或最接近真实的 expected-profit objective 选菜单；
   - 与 Cost Oracle 分开报告。

### 对比方法

- Nearest-L
- Cost-L heuristic
- CNN-Menu
- old CNN-SetMenuNet
- Expected-Profit Enumeration
- Service-Constrained Expected-Profit
- Cost Oracle
- Profit Oracle

## 9. 成功门槛

### 进入 Phase 09 Formal 的条件

Expected-Profit 或 Service-Constrained Expected-Profit 至少满足：

1. adjusted profit 或 service-constrained net profit 超过 Cost-L 和 CNN-Menu；
2. quit rate 不异常升高；
3. avg walk / avg IVT 没有明显恶化到不可解释；
4. Profit Oracle 能形成比 Cost Oracle 更合理的目标参考；
5. 3-seed pilot 方向一致，不能只靠单 seed。

### 如果 pilot 不支持

不要回到“强行调 CNN-SetMenuNet”。应输出诊断：

- 收益函数是否缺少 quit penalty；
- MNL 参数是否导致 outside option 过强或过弱；
- 价格范围是否让服务菜单缺乏吸引力；
- 候选点构造是否导致菜单缺少有效差异；
- route-cost feedback 是否与菜单目标使用的成本不一致；
- 当前仿真场景是否本身不支持有限菜单收益。

## 10. 论文叙事改写

旧叙事：

> CNN-SetMenuNet learns better set representations and therefore improves DRT menu design.

新叙事：

> Insertion-cost-based menu selection can be misaligned with realized system profit under passenger choice. We propose a choice-aware expected-profit menu optimization framework that explicitly accounts for passenger response, pricing, route-cost feedback, and opt-out risk.

CNN-SetMenuNet 在新叙事中的位置：

- 不是主方法；
- 是 insertion-cost representation learning 的代表；
- 用于证明“更好成本排序不必然带来更好运营收益”；
- 可以作为后续 ProfitAware learning 的结构候选，但不能作为当前正向贡献硬推。

## 12. 最终建议

现在不要再问：

> 怎样让 CNN-SetMenuNet 赢？

应该问：

> 为什么低插入成本菜单不能带来高收益？怎样定义一个真正和乘客选择及系统收益一致的菜单目标？

这条路线更稳，也更容易形成有解释力的 TR Part E 论文贡献。
